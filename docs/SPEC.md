# «Главное» — Финальное ТЗ, дизайн и распределение задач (v2)

> Единый итоговый документ: продукт → дизайн → функции и экраны → задачи по ролям → как всё собрать вместе.
> Это **источник истины по продукту**. Технические контракты — в `api-spec.yaml`, `data-model.md`, `design-tokens.json`.

## Часть A. Продукт

- **Суть:** «Главное не провалится». Планировщик для студентов, который сам пересобирает день вокруг важного и помогает понять, почему планы срываются.
- **Изюминка:** автоперенос невыполненного + перераспределение по приоритету с подтверждением; гарантия закрытия главного. Rule-based (free) → ИИ-умнее (paid).
- **Аудитория:** студенты и молодые люди; коммерческий продукт.
- **Язык:** **старт на английском**; другие языки позже; переключение в настройках; копирайт — на EN.
- **Платформа:** **Flutter** — iOS, Android, **веб**; плюс **лендинг-сайт** с умной кнопкой **[Download]** (определяет устройство → нужный стор/веб).
- **Монетизация:** **подписка $10/мес — основная** (покрывает ИИ: рекуррентный расход → рекуррентный доход). **Реклама — опционально и только на бесплатном тарифе**, ненавязчиво (на платном нет; не портим «спокойный» продукт); реклама **не** финансирует ИИ — ИИ только платным. Лимиты по тарифам; видео не self-host; код считает числа. *(Реклама даёт низкий ARPU, особенно в РФ; после 2022 сторовые платежи и западные рекламные сети для РФ ограничены — при таргете на Россию проверить RuStore / Yandex-рекламу.)*
- **ИИ:** через **Claude API** (бэкенд → endpoint Messages, ключ из Anthropic Console, оплата за токены, ключ только на сервере). Модели: **Haiku 4.5** ($1/$5 за млн токенов) на массовое, **Sonnet 4.6** ($3/$15) на сложное; мультимодаль — на фото (число КБЖУ всё равно из базы). Удешевление: prompt caching (−90% на повторный контекст) и Batch (−50%). *(Sonnet 3.5/3.6 — старое поколение, не использовать.)*
- **Цепляющие механики:** стрики (всё закрыто / выпил норму) + заморозка стрика; кольцо «всё закрыто»; два ритуала (утренний разбор + вечерний/предварительный на завтра-неделю); два тона (мягкий/жёсткий); wrapped-отчёты; лёгкий ввод; шеринг по веб-ссылке + «приведи друга».

## Часть B. Дизайн (итоговый)

### B1. Принципы
Контрастно и «дорого», характер — не дженерик; 1–2 тапа на действие; крупные тач-цели; читаемость на мелком; живые, но короткие микроанимации; единая компонент-библиотека; всё в выбранной теме.

### B2. Темы — каждая со своим шрифтом, дизайн-языком и настроением
Пользователь выбирает в настройках (+ «Авто»). Семантика (важное / срочно / готово) переопределяется под тему; жёсткий режим → акцент ember.

1. **Focus (тёплый тёмный, по умолчанию).** Палитра: фон `#141009`, поверхность `#241D11`, текст `#F6EFE1`, лайм `#D9F24B`, ember `#FF6A3D`. Шрифты: дисплей **Fraunces**, текст **Hanken Grotesk**. Язык: тёплый, редакционный, «уютный фокус».
2. **Calm (расслабление, цветопсихология).** Сине-зелёная приглушённая гамма низкой насыщенности: фон `#11171A`, поверхность `#18232A`, текст `#E8F0F0`, акцент `#6FB6A3`. Шрифты: дисплей **Newsreader** (мягкий серив), текст **Mulish** (округлый). Язык: воздушный, спокойный, много пространства.
3. **Black (OLED фулл-блек).** Фон `#000000`, поверхность `#0E0E0E`, текст `#FFFFFF`, акцент `#C8FF4D`. Шрифт: **Schibsted Grotesk** (резкий, технологичный). Язык: минимализм, максимальный контраст, премиально.
4. **White (чистый светлый).** Фон `#FFFFFF`, поверхность `#F5F4F1`, текст `#16130E`, акцент `#2B2A26` + `#5B7CFA`. Шрифты: дисплей **Instrument Serif**, текст **Geist**. Язык: редакционно-светлый, много воздуха.
5. **Contrast (доступность).** Максимальный контраст, крупный кегль. Шрифт: **Atkinson Hyperlegible** (создан под читаемость). Язык: жирно, однозначно.

### B3. Анимации — короткие (120–300 мс), снэппи; у каждой вкладки свой характер
- **Per-action:** чек — галочка + сжатие; кольцо — замыкание дуги + микропружина на 100%; стакан — заполнение волной; перенос задачи — карточка «уезжает» с тенью; стрик +1 — счётчик подпрыгивает; принятие разбора — карточки складываются; конфетти при «всё закрыто».
- **Per-tab характер:**
  - **Today** — энергичный «подъём» карточек + пружина кольца.
  - **Plan** — латеральное (свайп недель, блоки въезжают сбоку), «календарное».
  - **Health** — текучее/органическое (волна воды, заполнение колец, кросс-фейд сегментов).
  - **Diary** — спокойный, медленнее: мягкое появление снизу, рефлексивный ритм.
- **Глобально:** модалки — спринг снизу; смена вкладок — быстрый кросс-фейд. Все отключаемы в доступности.

### B4. Фирменные элементы
Кольцо «всё главное закрыто» (+стрик), стакан воды с тапом, карточка «Разбор», тумблер тона, иконка (+ альтернативные — premium).

### B5. Референс
Кликабельный HTML-макет «Сегодня» + 5 тем уже есть — дизайнер разворачивает их на все экраны (Часть C).

### B6. Тексты тона — стартовые примеры (EN; копирайт дорабатывается)
Мягкий — по умолчанию; жёсткий — опционально, гротескно-смешной, **без шейминга еды/тела**.

| Момент | Gentle | Harsh |
|---|---|---|
| Утренний разбор | "Yesterday left 3 loose ends — I tucked them into today around what matters." | "3 tasks ghosted you yesterday. I sorted them. Don't ghost them again." |
| Принять план | "Nice. Let's protect what matters today." | "Good. Now actually do them." |
| Всё главное закрыто | "Everything that mattered — done. Proud of you." | "Look at you, finishing things. Don't get cocky." |
| Просрочен дедлайн | "A deadline slipped. It happens — let's reschedule." | "Deadline's toast. Let's not make it a habit." |
| Напоминание про воду | "Quick sip? Your body will thank you." | "Drink some water. Hydrate or regret it." |
| Вернулся после пропуска | "You're back — that's what counts. One step at a time." | "Back already? Good. Less scrolling, more doing." |
| Разбор на завтра (вечер) | "Want tomorrow handled? I've got a plan ready." | "Plan tomorrow now, or wing it and panic. Your call." |

## Часть C. Функции и экраны

**Навигация:** 4 нижних вкладки — **Today · Plan · Health · Diary**; **профиль — аватар в левом верхнем углу** (не в таб-баре). Плюс **виджет** на домашний экран телефона (ближайшая пара/задачи/кнопка разбора).

### C1. Онбординг и вход (единый поток)
Splash → **единый онбординг**: 2–3 слайда ценности, плавно переходящие в настройку (вход e-mail/пароль + Google/Apple с согласием → интересы → импорт расписания → время разбора → тон → тема → базовые нормы) → старт. Один связный поток, без отдельных «карусели» и «визарда». (опц.) мини-тур подсказок при первом заходе на экраны.

### C2. Today
- Шапка (приветствие, дата) + **аватар-профиль слева сверху** + тумблер тона справа.
- Кольцо «главное X/Y» (замыкается на 100%) + стрик + ряд 7 точек (последние 7 дней).
- **Утренний разбор:** «вчера не закрыто N» + тон-строка + предложенные переносы (ИИ предлагает) + **[Принять]** **[Оставить как есть]** **[Поправить]**; **[Пересчитать]** 1×/день.
- **Главное сегодня:** 1–3 защищённые задачи (бейдж-щит, чек).
- **Дальше по дню:** хроно-список (события расписания + задачи, чек/приоритет/время, drag&drop, свайп).
- **FAB [+]** → создать задачу (естественный ввод, тип, приоритет/«главное» ≤1–3, повтор, напоминания).
- Состояния: всё закрыто → кольцо + конфетти; просрочка → ember.

### C3. Разбор завтра / недели (новый сценарий)
По расписанию (вечером) — полноэкранное предложение или маленькое тапабельное уведомление. Тап → ИИ-план на завтра (или неделю), **несколько вариантов раскладки**; **[Принять вариант]** / **[Отказаться → вручную]**. Вручную: голосом создать/перенести, перетаскивание задач.

### C4. Plan
- Недельная полоса (свайп недель) + дневная лента по времени; карточка экзамена/дедлайна с обратным отсчётом (закреплена).
- Переключатель День/Неделя/Месяц; **[+]** добавить.
- **Добавить событие/дедлайн/экзамен:** **готовые варианты** — предметы (пресет + свой) и типы (лекция/семинар/лаба/экзамен/дедлайн), дата/время/аудитория, повтор, напоминание.
- **Импорт расписания:** конструктор-шаблон + **[Клонировать неделю]** (основной), фото/скриншот + OCR, вставка текста, голос, (опц.) подписка ICS. Везде — экран подтверждения с правкой.
- **Долгосрочные цели:** Месяц / Год / 5 лет / 10 лет, подзадачи, прогресс, связь с дневными задачами.

### C5. Health (хаб: Food · Water · Workouts · Sleep)
- **Food [Ф1]:** карточка дня (ккал/норма, Б/Ж/У, бары **Сахар** и **Клетчатка**), карточка Баланс (+мягкая подсказка). Приёмы: Завтрак/Обед/Ужин + всегда доступный **«+ перекус»** (добавляет к итогам без жёсткого слота). «Съел по плану» — 1 тап. Ввод: штрихкод / фото (редактируемая оценка, лимит 3–4/день) / поиск / рецепт из ингредиентов / ресторан-меню / голос → результат с КБЖУ из базы + правка граммов. План меню Месяц→Неделя→День→Блюдо (+рецепт/видео/**[ИИ-помощь]**); **[Собрать ИИ]** (компонует из базы, числа считает код). Список покупок (готов к доставке, Ф3). Мягкий режим (скрыть калории).
  - **Баланс рациона (правила):** план/день сбалансирован, если по индивидуальным нормам пользователя — калории в коридоре цели; белок не ниже нормы; **клетчатка** ≥ цели (~25–30 г/день или ~14 г на 1000 ккал); **сахар** ниже потолка (ориентир ВОЗ: свободные сахара <10%, в идеале <5% калорий); есть овощи/разнообразие. Иначе — мягкая конкретная подсказка («добавь клетчатки/овощей», «белок низковат»). Точные пороги финализирует нутрициолог; формулировки — без шейминга.
- **Water [Ф1]:** стакан (тап = выпил, анимация), настройка объёма, норма (база + еда позже), график, мягкие предупреждения.
- **Workouts [Ф2]:** план неделя/месяц (циклы); тренировка (упражнения: техника/видео/вес/отдых); режим «тренёр» (видео, таймер, голос, **[Сделал по плану]**); **осанка** (упражнения + напоминания «выпрямись»); дыхание/медитации (сессии аудио/видео).
- **Sleep [Ф2]:** карточка ночи; **[Ложусь]** / **[Проснулся]**; тумблер импорта из **Health Connect / Apple Health** (читаем готовый сон, без ручного ввода); график.

### C6. Diary
- **Рефлексия:** настроение; **план vs факт**; **факт дня** — отметить, что реально было («был в клубе»); **«что пошло не так»** связывает пропуски с причинами (чипы: соцсети / гулял / устал + заметка) и показывает паттерны; инсайт (rule-based; ИИ глубже — paid); поле «что интересного»; **[Сохранить день]**. *(Авто-замер соцсетей — только Android с разрешением Usage Access; иначе самоотчёт.)*
- **Wrapped-отчёт [Ф1]:** Неделя/Месяц — кольцо, стрик, закрыто X, топ-срывы и причины, сводка вода/сон, «неделя одним абзацем», **[Поделиться]** (картинка).

### C7. Профиль (аватар слева сверху)
- Профиль: цифры, стрик, **заморозка стрика**.
- Настройки: тема (5 + авто, превью), тон по умолчанию, язык, размер шрифта, уведомления + тихие часы, иконка, мягкий режим питания.
- **Шеринг:** **[Поделиться планом]** → **view-only веб-ссылка** (друг открывает в браузере, **приложение не нужно**) + мягкий CTA «get the app»; «Поделились со мной» + **[Скопировать себе]** (в приложении). Реферал **[Привести друга]**, **[Отзыв]**, соцсети.
- Подписка/пейволл (free vs premium), FAQ, поддержка, о приложении.

### C8. Фокус-сессии и со-учёба [Ф2–Ф3]
- **Фокус-сессия:** пресеты 25/5, 50/10, 52/17, 90/20, **+ фирменный 67/15** (хайповый — необычное число как фишка, ~в рамках ультрадианного цикла), свой. Экран: таймер, фаза, **[Пауза] [Стоп]**; трение при выходе («ты в сессии?») без слежки за приложениями; засчитывается в фокус-время/стрик.
- **Со-учёба:** группы/друзья — видно, кто в сессии и сколько; **[Начать совместную]** (инвайт, общий таймер); недельный лидерборд часов.

## Часть D. Распределение по ролям (каждому — своя задача)

### D1. Дизайнер (UX/UI + визуал)
- **[MVP]** Потоки (онбординг, день, разборы, рефлексия); макеты 4 вкладок + профиль-слева + виджет; компонент-библиотека; темы Focus/Black/White (палитра+шрифт+язык по B2); анимации (B3, в т.ч. per-tab); иконка.

### D2. Мобильный разработчик (Flutter)
- **[MVP]** Каркас + навигация (4 таба, профиль-слева, хаб Health); система тем (переключение/сохранение/авто); анимации; offline-first (SQLite/Drift + синхрон); локальные уведомления (лимиты iOS, тихие часы); экраны Today/Plan/Diary; конструктор расписания + фото/OCR/вставка/голос; **виджет** (iOS WidgetKit / Android App Widgets); разбор утренний + вечерний (UI вариантов, голос, drag).
- **[Ф1]** Экраны питания; сканер штрихкода; ввод фото/голос.
- **[Ф2]** Интеграции Health Connect/Apple Health; плеер видео/аудио (CDN).
- **[Ф3]** Шеринг-вью / deep links; «поделились со мной» + копирование.
- Веб-сборка (Flutter Web).

### D3. Бэкенд-разработчик
- **[MVP]** Аккаунты (e-mail/пароль + Google/Apple) + синхрон; **модель данных** (User; Item + типы + приоритет; RecurrenceRule + Exception; Reminder; Subject; Goal; DayLog; TimelineEntry/Факт; Streak + freeze; WaterLog; Product/Recipe/Dish/MealPlan; Workout/Exercise; ShareLink…); **rule-движок перераспределения** (утро/вечер/неделя, варианты); стрики/счётчики; хранилище медиа.
- **[Ф1]** Интеграция базы питания (OFF/Nutritionix); расчёт КБЖУ + сахар + клетчатка.
- **[Ф3]** **Шеринг:** генерация **публичной view-only веб-страницы/PDF** с контролем доступа (без доступа к аккаунту) + CTA; копирование плана.
- Сквозное: приватность (Часть F); лимиты ИИ-действий по тарифам.

### D4. ИИ / ML-инженер
- **[paid]** Умное перераспределение (паттерны, реалистичность; несколько вариантов).
- **[Ф1]** Сборка меню из базы (числа считает код); фото-распознавание (vision → блюдо/порции → правка → расчёт из базы; лимит 3–4/день) + ресторан-меню/рецепт.
- **[Ф2/Ф3]** ИИ-разбор дня (причины, советы).
- Сквозное: **Claude API** — Haiku 4.5 на массовое, Sonnet 4.6 на сложное, мультимодаль на фото; prompt caching + Batch для удешевления; ключи на бэкенде. Бэкенд-эндпоинты вызывают Claude и отдают JSON приложению.

### D5. Контент / данные
- **[MVP]** Пресеты предметов и типов (расписание/дз/экзамены).
- **[Ф1]** База продуктов и блюд (источник истины); проверенные КБЖУ + сахар + клетчатка.
- **[Ф2]** Упражнения (техника/видео); осанка; сессии дыхания/медитаций.
- Тексты тона (мягкий/жёсткий) — отдельный лист копирайта (EN).

### D6. QA
- Тест-планы: движок повторов и перераспределения (утро/вечер/неделя/варианты), стрики + заморозка, синхрон + оффлайн, импорт расписания, расчёты питания, шеринг-доступ, приватность; кросс-устройства (iOS / Android / веб).

### D7. Рост / маркетинг
- Лендинг + умная **[Download]**; реферал/отзыв/соцсети; wrapped как шерящийся контент; ASO; позиционирование «for students — so the important stuff never slips».

## Часть E. Как это собирается вместе (сборка/интеграция)

**Зависимости (что от чего):**
- **Дизайн-система + модель данных = фундамент.** Делаются первыми — на них опираются все.
- Мобайл строит экраны поверх **API бэкенда** (договорённые контракты) и дизайн-системы.
- ИИ работает **через бэкенд** (не из клиента) и поверх **базы питания** (контент).
- Шеринг-веб-страница и веб-версия — общий веб-слой.

**Контракты интеграции:**
- Бэкенд публикует API (задачи / разбор / стрики / питание / шеринг) — мобайл и ИИ его потребляют.
- Синхрон: локальная БД ↔ бэкенд (offline-first, last-write-wins).
- ИИ: бэкенд-эндпоинты «собери меню», «разбор дня», «варианты плана»; числа КБЖУ — из базы, не из модели.

**Порядок по фазам (Definition of Done каждой):**
1. **MVP:** дизайн-система + модель данных → аккаунты/синхрон → экраны Today/Plan/Diary → rule-разбор (утро + вечер + варианты) → импорт расписания → стрики/заморозка → онбординг → темы Focus/Black/White → виджет. *Готово = бесплатное приложение, которым пользуешься ежедневно, без ИИ.*
2. **Ф1:** база питания + модуль Food (КБЖУ/сахар/клетчатка, штрихкод/фото/рецепт/ресторан, ИИ-сборка) + Water + список покупок + wrapped + подписка/пейволл + ИИ-надстройки. *Готово = платный контур работает.*
3. **Ф2:** Workouts + осанка + дыхание + Sleep + Health-интеграции + тема Contrast + фокус-сессии (вкл. 67/15).
4. **Ф3:** шеринг по веб-ссылке + «поделились со мной» + копирование + со-учёба + интеграция доставки. Гардероб — позже.

**Как соединить работу людей:** общий репозиторий и доска задач; единые **API-контракты и дизайн-токены** как «стыки»; еженедельная сборка; на каждой фазе — приёмка по Definition of Done выше.

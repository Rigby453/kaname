# Kaizen — Домашний виджет (спецификация)

> Источник истины по домашнему виджету. Связан со SPEC §C (Today/Plan), MASCOT.md (Kai),
> UX-LAYOUT.md (иерархия), design-tokens.json (цвета тем). Решение — decisions.md → **ADR-042**.
> Утверждено пользователем 2026-06-19: **акцент на задачи**, компактно но информативно,
> стрик рядом без акцента, Kai выглядывает с эмоцией по частоте захода, все размеры + выбор.

---

## 1. Задача виджета (одна)

Ответить за 1–2 секунды, **не открывая приложение**: «что у меня сейчас/дальше» — и затащить
обратно через Kai. Поэтому наполнение по приоритету внимания, не «всё подряд».

**Акцент — на ЗАДАЧИ** (ближайшие пары/дела со временем). Стрик и Kai — вторичны.

---

## 2. Что показывать (иерархия)

| Приоритет | Элемент | Где появляется |
|---|---|---|
| 1 | **«Дальше по дню»** — ближайшие 1–3 пункта (время + название + иконка типа task/event/exam) | все размеры (кол-во растёт с размером) |
| 2 | **Кольцо/счётчик «главное X/Y»** — ядро продукта (3 главные задачи) | средний и больше |
| 3 | **Стрик** 🔥N — компактно, в углу, **нейтральным цветом** (не accent) | все размеры |
| 4 | **Kai** — выглядывает из угла/края, эмоция по состоянию (см. §4) | все размеры |
| 5 | **Кнопка `+`** (добавить задачу) | большой |

**Не показываем** воду/еду/дневник — это размывает фокус. Возможны отдельные тематические
мини-виджеты позже; основной виджет = **план дня**.

Если задач/событий на сегодня нет → строка «Сегодня свободно» + Kai (idle/away),
тап ведёт на добавление.

---

## 3. Размеры (все поддерживаем, выбор за пользователем)

Виджет **адаптивный**: одно семейство, раскладка зависит от размера (Android 12+ responsive
layouts / iOS WidgetFamily). Пользователь выбирает размер при установке/ресайзе.

```
МАЛЫЙ 2×2                СРЕДНИЙ 4×2                     БОЛЬШОЙ 4×4
┌───────────┐            ┌────────────────────┐          ┌────────────────────┐
│ 14:30 (k) │            │ Дальше      (Kai)🔥7│          │ Сегодня     (Kai)🔥7│
│ Лекция    │            │ 14:30 Лекция        │          │ ◓ 2/3 главных       │
│           │            │ 16:00 Эссе   ⚑      │          │ ──────────────────  │
│ ◓2/3  🔥7 │            │ ◓ 2/3 главных       │          │ 14:30 Лекция        │
└───────────┘            └────────────────────┘          │ 16:00 Эссе    ⚑     │
  1 пункт + кай-уголок      2 пункта + кольцо + стрик     │ 18:30 Зал           │
                                                          │ +2 ещё          [+] │
                                                          └────────────────────┘
  k = Kai выглядывает из угла (маленький). Стрик нейтральным цветом.
```

- **Малый 2×2:** 1 ближайший пункт (время+название) крупно + «X/Y» мелко + стрик + Kai-уголок.
- **Средний 4×2:** 2 ближайших пункта + «X/Y главных» + стрик + Kai выглядывает справа сверху.
- **Большой 4×4:** мини-расписание (3–4 пункта со временем) + «X/Y» + стрик + Kai + кнопка `+`.

(Опционально позже: отдельные «стилевые» варианты — напр. «Kai крупно + стрик» как дашборд
ретеншена. База — task-focused выше.)

## 4. Kai в виджете (выглядывает, эмоция по частоте захода)

Kai **не по центру** — выглядывает из угла/края («peek»), не перекрывает задачи (MASCOT.md §6:
никогда не перекрывает контент). Эмоция (PNG-кадр, §6) выбирается по состоянию дня + давности
захода:

| Условие | Эмоция Kai (MASCOT.md §5) |
|---|---|
| Заходил сегодня + все главные закрыты | **success** (горд) |
| Заходил сегодня + есть прогресс/в норме | **neutral** |
| Есть просрочка / отстаёт | **anxious** |
| Не заходил ≥ 2 дней | **away** (глаза-«нитки», осевший, грустно выглядывает) |
| Тон harsh (любое состояние) | вариант с глазами → **ember** |

«Дней с последнего захода» считается при само-обновлении виджета (§7) из сохранённого
timestamp последнего открытия. Глаза Kai = accent активной темы (MASCOT.md §3).

**Тап по Kai → открыть приложение.** Тап по пункту → открыть этот день/задачу. Тап `+` → добавить.

## 5. Цвет и тема

- Виджет **читает активную тему** (accent / bg=surface / text / textMuted) и красится под неё —
  не захардкожен в Focus. Цвета приходят через data-bridge (§8).
- **Дисциплина акцента (UX-LAYOUT §6):** accent — только глаза Kai и, опционально, маркер
  «главного»/кольцо. Время и названия задач — обычный текст; стрик — `textMuted` (нейтрально,
  без accent). Ember — только просрочка.
- Фон — surface темы, скругление 24dp, hairline-граница (как карточки приложения).

## 6. Kai = заранее отрендеренные PNG (не Flutter в виджете)

Виджеты рендерятся системой (Android RemoteViews / iOS WidgetKit), Flutter/CustomPainter/Rive
там недоступны. Поэтому Kai в виджете — **набор PNG-кадров**, отрендеренных из существующего
`KaiMascot` CustomPainter скриптом (golden-render в PNG), без дизайнера:

- Эмоции: **neutral / success / anxious / away** (4 базовых; harsh — отдельный вариант глаз).
- Под темы: цвет глаз = accent темы. Чтобы не плодить ассеты — рендерим тело нейтральным +
  накладываем accent программно (RemoteViews `setColorFilter` / WidgetKit `.foregroundColor`),
  либо рендерим компактный набор эмоций × тем. Финальный набор определяется при сборке.
- Плотности Android (mdpi…xxxhdpi) и масштаб iOS — из одного исходного рендера.

## 7. Обновление (чтобы Kai «грустнел» без открытия)

- **На событиях:** app resume + изменение задач (как сейчас) → push свежих данных.
- **По таймеру (новое):** периодический рефреш, чтобы эмоция Kai и «дальше по дню» менялись,
  даже если приложение не открывали.
  - Android: `updatePeriodMillis` (мин. ~30 мин) или WorkManager (раз в день/час).
  - iOS: `Timeline` с запланированными записями на будущее (бюджет ~40–70 обновлений/сутки).
- Данные кэшируются (Android SharedPreferences / iOS App Group UserDefaults) — при закрытом
  приложении виджет показывает последние сохранённые значения.

## 8. Data-bridge (что приложение передаёт виджету)

Расширяем текущий мост (сейчас только `main_progress` + `streak`). Поля (snake_case):

| Поле | Тип | Описание |
|---|---|---|
| `next_items` | JSON-массив (до 4) | `[{time, title, type}]` ближайшие пункты дня |
| `main_done` / `main_total` | int | прогресс главных задач |
| `streak` | int | текущая серия |
| `kai_emotion` | string | neutral/success/anxious/away (вычислено по §4) |
| `is_harsh` | bool | тон (для варианта глаз ember) |
| `theme_accent` / `theme_bg` / `theme_surface` / `theme_text` / `theme_text_muted` | hex | цвета активной темы |
| `last_opened_at` | ISO | для пересчёта «дней без захода» при таймерном рефреше |

Источник: `ItemsDao` (ближайшие по `scheduledAt` от now), `StreakDao`, prefs (тема, тон,
last-open). Реализация — `app/lib/services/widget/widget_service.dart`.

---

## 9. Платформы

| Платформа | Статус | Примечание |
|---|---|---|
| **Android** | основная, собирается и проверяется | RemoteViews, responsive layouts (2×2/4×2/4×4) |
| **iOS** | код пишем, **сборка/проверка — нужен Mac** | WidgetKit extension + App Group; Lock Screen/StandBy виджеты — бонус позже |
| **Web / PWA** | нет ОС-виджетов | максимум ярлык/иконка; «дашборд» — отдельная веб-задача, не виджет |
| **Windows / desktop** | вне scope | Windows Widgets Board — нишево; не делаем сейчас |

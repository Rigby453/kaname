## Вариант A — «Компактная статус-строка» (РЕКОМЕНДУЮ)

Идея: Kai остаётся крупным и заметным (как ты любишь), но числовой статус (кольцо + стрик)
схлопывается в одну компактную строку, а приветствие/дата ужимаются. Задачи поднимаются на ~300px выше.

```
┌─────────────────────────────────────────┐
│  Понедельник, 27 июня        [🙂 тон ▾]  │  ← дата мелко; тон уезжает в меню Kai (см. ниже)
│                                          │
│                 (  Kai  )                │  ← Kai 104dp, крупный, по центру
│              «Главное сделано?»          │  ← пузырь
│                                          │
│  ◔ 2/3 главных   ·   🔥 12   ·   ❄ 3     │  ← НОВАЯ компактная строка-статус (вместо
│                                          │     большого кольца + отдельной строки стрика)
├─────────────────────────────────────────┤
│  [ Утренний разбор: перенести 4 → ... ]  │  ← hook на месте, сразу под статусом
│                                          │
│  Задачи на сегодня                       │  ← задачи видны уже на первом экране
│  □ ...                                    │
```

Что меняется:
- Большое кольцо `ProgressRing` → маленькое инлайн (◔ 2/3) внутри статус-строки. Кольцо
  не исчезает, просто перестаёт занимать пол-экрана.
- `StreakRow` (🔥 + 7 точек) → 🔥12 ·  ❄3 в той же строке (точки недели можно открыть тапом).
- Приветствие 40sp → дата обычным размером (приветствие дублирует пузырь Kai — убираем
  гигантский заголовок, оставляем тёплый пузырь).
- Тумблер тона уезжает из шапки в меню Kai (тап по Kai) — это же и есть «упростить кнопки Kai».
- Вечерний разбор — рендерить только когда реально есть что показать.

Плюсы: низкий риск, большой выигрыш по месту, Kai остаётся героем. Минусы: кольцо меньше
(но оно и так чисто декоративное — точное число важнее размера).

## Вариант B — «Сворачивающаяся шапка при скролле» (красиво, дороже)

Шапка (Kai + статус) живёт в `SliverAppBar`: при прокрутке к задачам сворачивается в тонкую
полоску (мини-Kai + ◔2/3 + 🔥12). Наверху списка — полный вид, при скролле — компактный.

```
Вверху списка:            После скролла (закреплённая полоска):
┌──────────────────┐      ┌──────────────────────────────┐
│   ( Kai 104 )    │      │ (Kai 32) ◔2/3 · 🔥12   Сегодня │
│  «...»           │      └──────────────────────────────┘
│ ◔2/3 ·🔥12 ·❄3   │      □ задача
└──────────────────┘      □ задача
```

Плюсы: и крупный Kai, и максимум места под задачи при работе со списком. Минусы: больше
кода (Sliver-перестройка экрана), чуть выше риск багов вёрстки.

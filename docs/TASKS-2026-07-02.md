## 1. ИИ контекст-онбординг + ИИ-быстрое добавление задачи — ГЛАВНОЕ
**Цель:** дать ИИ КОНТЕКСТ о пользователе с самого начала (пробный премиум разблокирует фичу — её
надо ВЫДЕЛИТЬ, чтобы сразу попробовали) и максимально удобный ввод.

### 1a. Онбординг-брейндамп → стартовый план
- **[ ]** Экран после регистрации: пользователь рассказывает о себе — **текстом ИЛИ голосом** (в
  приложении уже есть голосовой ввод, напр. при добавлении задачи — переиспользуем его, отдельная
  STT-инфраструктура не нужна).
- **[ ]** Вопросы показываем **прямо во время диктовки/печати** (чтобы не забыл): промпт-вопрос
  сверху + поле/плашка ввода под ним, вопросы идут по одному или списком-подсказкой рядом с полем.
  UX обыграть — решаю сам, критерий «удобно, не теряется мысль».
- **[ ]** Backend `POST /api/v1/ai/onboarding-plan` (premium): вход = ответы + дата/таймзона; выход =
  структурированный план (goals[], tasks[] c priority/scheduledAt/deadline, опц. food_prefs). JSON,
  thinkingBudget:0, языковая directive, провайдер-агностик.
- **[ ]** Превью «вот твой план» → пользователь принимает/правит → создаётся в Drift + синк
  (не автосейв молча — подтверждение).
- **Черновик вопросов (на утверждение):** чем занят (учёба/работа/проект); что важное не упустить в
  ближайшие дни; дедлайны/даты; что часто откладываешь; (опц.) следишь ли за питанием/водой/сном;
  плотный день или растянутый.

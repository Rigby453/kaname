# Кнопки разбора в «Сегодня» — что есть и что под вопросом

> Заметка-памятка (создана 2026-06-25). Цель — зафиксировать, какие кнопки уже
> работают в экране «Сегодня», как они соотносятся со SPEC, и какие фичи **под
> вопросом** (решить в следующей сессии). Чтобы не забыть.

Файлы:
- `app/lib/features/today/widgets/morning_review_card.dart` — утренний разбор
- `app/lib/features/today/widgets/evening_review_card.dart` — вечерний разбор
- `app/lib/features/today/widgets/review_engine.dart` — общая логика построения вариантов
- `app/lib/features/today/widgets/review_variant_card.dart` — карточка варианта раскладки

## ✅ Что уже работает

### Утренний разбор (Morning review)
Карточка появляется, **если есть просроченные невыполненные задачи** с прошлых дней.

| Элемент | Что делает |
|---|---|
| Кнопка **«Разобрать»** (`today.review_btn`) | Открывает шит утреннего разбора. |
| Иконка **AI-подсказки** (✨ `auto_awesome`) | Премиум. Запрашивает у бэкенда tone-aware сообщение (`/ai/morning-message`), мягкое/строгое по тону. Не премиум → апселл. |
| В шите: **«Перенести всё на сегодня»** (`today.move_all_today`) | Переносит все просроченные задачи на сегодня, распределяя по разным свободным слотам (не в стак). |
| В шите: **умные варианты раскладки** (rule-based, бесплатно) | `buildVariants()` строит несколько раскладок по приоритету в свободные слоты 08:00–22:00. Тап «Применить» → `applyVariant()`. |
| В шите: кнопка **«AI умнее»** (`today.ai_smarter_plan`) | Премиум. `/ai/redistribute` (target = сегодня) → 2–3 AI-варианта плана. Не премиум → апселл. |
| Per-task: **«На сегодня»** + **пропуск** (➖) | Перенести одну задачу на сегодня / отметить пропущенной. |

### Вечерний разбор (Evening review, SPEC C3)
Карточка появляется **после 17:00**. Помогает перенести сегодняшнее незакрытое на завтра.

| Элемент | Что делает |
|---|---|
| Кнопка **«Спланировать завтра»** (`today.plan_tomorrow_btn`) | Открывает шит вечернего разбора. |
| В шите: **«Перенести всё на завтра»** (`today.move_all_tomorrow`) | Переносит все сегодняшние незакрытые задачи (только `type=task`) на завтра по разным слотам. |
| В шите: **умные варианты раскладки** (rule-based, бесплатно) | Те же `buildVariants()`, но на завтрашний день. |
| В шите: кнопка **«AI умнее»** (`today.ai_smarter_plan`) | Премиум. `/ai/redistribute` (target = завтра) → AI-варианты. |
| Per-task: **«На завтра»** + **пропуск** (➖) | Перенести одну задачу на завтра / отметить пропущенной. |

> Итого: «умные варианты переноса» и «AI умнее», про которые ты спрашивал, — это
> **rule-based варианты раскладки** + **AI-варианты** (`/ai/redistribute`). Они есть
> и в утреннем, и в вечернем разборе.

## ❓ Под вопросом — РЕШИТЬ в следующей сессии

Эти пункты есть в SPEC, но **не реализованы как отдельные кнопки**. Решить, нужны ли.

### 1. ❓ «Пересчитать» (SPEC C2, строка ~75: `[Пересчитать] 1×/день`)
- **Что по спеке:** маленькая бесплатная кнопка в утреннем разборе, доступна **1 раз в
  день**, перекатывает (re-roll) варианты раскладки заново.
- **Текущий статус:** отдельной кнопки НЕТ. По сути её роль закрывают «умные варианты» +
  «AI умнее» (они и так показывают раскладки).
- **Оценка:** ценность отдельной кнопки маргинальная. Реализация ~10 минут, если решим, что
  нужна (добавить кнопку «Пересчитать» + лимит 1×/день в шапку шита утреннего разбора).
- **РЕШЕНИЕ:** _не принято._ → нужна ли отдельная кнопка «Пересчитать»? (да / нет)

### 2. ❓ «Разбор недели» (SPEC C3, строки ~81–82: «Разбор завтра / **недели**»)
- **Что по спеке:** вечернее предложение плана не только на завтра, а **на всю неделю**,
  с несколькими вариантами раскладки.
- **Текущий статус:** «завтра» сделано (вечерний разбор). «На всю неделю» — НЕ сделано.
- **Оценка:** тяжёлая фича. Потребность во многом уже покрыта вечерним разбором «на завтра»
  + экраном «План». Низкий приоритет.
- **РЕШЕНИЕ:** _не принято._ → делаем недельный разбор сейчас / откладываем / режем из SPEC?

### 3. ❓ Согласованность с SPEC
- В STATUS.md строка ~19 этот пункт помечен как открытый: «`[ ] «разбор недели»/«Пересчитать»
  из SPEC (уточнить нужность — это перепланирование дня, НЕ КБЖУ)». Закрыть после решения.

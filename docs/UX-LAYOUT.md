### Глобальная нижняя навигация / каркас (файл: app/lib/core/router/scaffold_with_nav_bar.dart)

Это общая оболочка `ScaffoldWithNavBar`, в которую вложены все 4 основных экрана. Она даёт ЕДИНЫЙ AppBar и нижнюю навигацию; вложенные экраны (Today, Plan) свой AppBar НЕ рисуют — только тело и FAB. Каркас адаптивный: до 600px — мобильный макет (нижний бар), от 600px — широкий (боковая рейка NavigationRail, нижнего бара нет).

**Нижний таб-бар (мобильный, <600px):** Material 3 `NavigationBar`, высота 64dp, фон = surface, без тени, сверху тонкая линия-разделитель (1dp). Активный таб подсвечивается «пилюлей» (indicator, accent с прозрачностью 0.15). У каждого таба иконка + подпись (всегда видна). 4 таба слева направо:
1. **Today** (nav.today) — иконка солнце (`wb_sunny_outlined` / активная `wb_sunny`).
2. **Plan** (nav.plan) — иконка календарь (`calendar_today_outlined` / `calendar_today`).
3. **Health** (nav.health) — иконка сердце (`favorite_border` / `favorite`).
4. **Diary** (nav.diary) — иконка книга (`menu_book_outlined` / `menu_book`).

Повторный тап по активному табу возвращает ветку к её корневому маршруту. Состояние каждой вкладки сохраняется (StatefulShellRoute).

**Верхняя шапка (AppBar) каркаса — мобильный:**
- **Слева:** кнопка-аватар профиля (`ProfileAvatarButton`) — круглый аватар (accent-заливка, иконка человечка), тап → переход на `/profile`. Профиль — НЕ таб, он всегда здесь слева сверху.
- **По центру:** заголовок = название активного таба (Today/Plan/Health/Diary).
- **Справа:** контекстные действия — показываются ТОЛЬКО на табе Plan (currentIndex == 1):
  - иконка **флажок** (`flag_outlined`) → переход на экран «Цели» (`/goals`).
  - иконка **загрузка файла** (`upload_file_outlined`) → открывает нижнюю шторку «Импорт расписания» для выбранного дня.
  На остальных табах справа пусто.

**Широкий макет (≥600px):** вместо нижнего бара — вертикальная боковая рейка `NavigationRail` слева с теми же 4 пунктами и подписями; профиль-аватар вынесен в верх рейки (leading). AppBar сверху без leading, заголовок слева (не по центру), те же действия Plan справа. Тело — справа от рейки.

**FAB:** в самом каркасе FAB нет — он определяется внутри каждого экрана.

### Экран Today (файл: app/lib/features/today/today_screen.dart)

- **Назначение** — главный экран «сегодня»: маскот Kai, кольцо прогресса, серия (streak), карточки разбора и список задач дня.
- **Шапка (AppBar):** своего AppBar нет — используется общий AppBar каркаса (профиль слева, заголовок «Today» по центру, справа пусто). Внутри тела сверху — собственная «Kai-шапка».
- **Тело сверху вниз** (вертикальный скролл-список, отступы 24dp по бокам):
  1. **Kai-шапка:** строка с приветствием по времени суток (слева, крупный заголовок) + дата под ним; справа в той же строке — **тумблер тона** (текстовая кнопка с иконкой: `bolt` «harsh» / `spa_outlined` «gentle», переключает строгий/мягкий тон).
  2. Под строкой — **маскот Kai** по центру (104dp, с анимацией «подпрыгивания» каждые 6–10с), под ним **речевой пузырь** с контекстным сообщением (если Kai отключён в настройках — вместо пузыря просто серый текст по центру).
  3. **Кольцо прогресса** (`ProgressRing`) по центру — доля выполненных главных задач.
  4. **Строка серии** (`StreakRow`) — счётчик streak.
  5. **Карточка утреннего разбора** (`MorningReviewCard`) — появляется при наличии просроченных задач из прошлых дней.
  6. **Карточка вечернего разбора** (`EveningReviewCard`) — появляется после 17:00.
  7. **Секция «Просрочено»** (`OverdueSection`) — вверху списка задач, ember-акцент.
  8. **Список задач дня** (`TaskList`) — секция `priority='main'` оформлена как **hero** (accent-разделитель + заголовок-счётчик); у задач — чек-лист подзадач (ADR-048).
  9. ~~**Секция «Привычки сегодня»** (`HabitsTodaySection`)~~ — **УБРАНА из Today** (упрощение геймификации). Виджет в коде остался, но не рендерится.
- **Кнопки и контролы:**
  - Тумблер тона — верх-право Kai-шапки (см. выше).
  - Тап по маскоту Kai — успокаивает его до нейтральной эмоции.
  - Плавающие кнопки снизу-справа (FAB-зона) — см. ниже.
- **FAB (плавающая «+»):** ДА. В правом-нижнем углу — круглый FAB с иконкой `+` (без подписи), тап открывает нижнюю шторку добавления задачи (`showAddTaskSheet`) на сегодня. **Слева от него** — малая круглая кнопка **отмены** `↩` (`undo`), видна ТОЛЬКО когда есть что отменить; тап откатывает последнее действие и показывает снэкбар-подтверждение. **Позиция FAB настраивается** (`fabPositionProvider.fabLocation`) — пользователь может менять расположение плавающей кнопки в настройках.
- **Нижние шторки / диалоги:** нижняя шторка «Добавить задачу» (по FAB `+`); снэкбар после отмены.
- **Свайпы/жесты:** свайпы по задачам реализованы внутри виджета `TaskList` (skip/done — в дочернем файле, не в этом экране). На уровне экрана — вертикальный скролл.
- **Поверх всего экрана:** прозрачный слой `CelebrationOverlay` (анимация-«поздравление» при выполнении всех задач).
- **Планшет (≥600px):** две колонки — слева Kai-шапка + кольцо + серия + карточки разбора; справа список задач (просрочка, задачи). FAB тот же. *(Привычки убраны и здесь.)*

### Экран Plan (файл: app/lib/features/plan/plan_screen.dart)

- **Назначение** — планировщик с переключением вида День / 3 дня / Неделя / Месяц / Год; таймлайн или сетка времени по выбранному дню.
- **Шапка (AppBar):** своего AppBar нет — общий AppBar каркаса. Именно на табе Plan в нём справа появляются 2 иконки: **флажок** → «Цели» (`/goals`) и **загрузка файла** → шторка «Импорт расписания».
- **Тело сверху вниз (мобильный):**
  1. **Тулбар** в одну строку (слева направо): компактный **выпадающий список вида** `[День ▾]` (`_ViewDropdown`, пункты: День/3 дня/Неделя/Месяц/Год) → **дата** (тап открывает системный DatePicker, со стрелкой ▾, занимает свободное место) → кнопка **«Today»** (только если выбран не сегодня, текстовая, справа от даты) → иконка **поиск** (`search`/`search_off`, тогглит строку поиска) → **тумблер раскладки** (`_LayoutToggleButton`: список ↔ сетка времени; иконки `grid_on`/`view_agenda_outlined`; только для День/3 дня/Неделя).
  2. **Строка поиска** (`_SearchField`) — разворачивается под тулбаром при включённом поиске; поле с иконкой лупы и крестиком очистки.
  3. Тонкий разделитель.
  4. **Содержимое вида** (занимает остаток):
     - День: раскрывающийся недельный календарь (`ExpandableWeekCalendar`, тянешь вниз — раскрывается месяц) → закреплённая ember-карточка ближайшего экзамена/дедлайна (`PinnedExamCard`) → таймлайн дня (`DayTimeline`) или сетка времени (`DayTimeGrid`).
     - 3 дня: карточка экзамена → сетка `ThreeDayTimeGrid` или агенда `ThreeDayAgenda`.
     - Неделя: раскрывающийся календарь (в списочной раскладке) → карточка экзамена → `WeekAgenda` / `WeekTimeGrid`.
     - Месяц: `MonthView` (календарная сетка с точками на днях с задачами).
     - Год: `YearView`.
- **Кнопки и контролы:**
  - Переключатель вида (выпадающий `[День ▾]` на мобильном; на планшете — широкий `SegmentedButton` из 5 сегментов, при нехватке ширины сворачивается в тот же дропдаун).
  - Дата (тап → DatePicker), «Today» (сброс на сегодня), поиск (тоггл), тумблер раскладки список/сетка.
  - «Цели» и «Импорт» — НЕ в теле экрана, а в общем AppBar справа (см. шапку).
- **FAB (плавающая «+»):** ДА. Круглый FAB `+` снизу-справа → открывает шторку добавления задачи (`showAddTaskSheet`) на выбранный день. Позиция FAB настраивается (`fabPositionProvider`). Кнопки undo здесь нет (в отличие от Today).
- **Нижние шторки / диалоги:** системный DatePicker (по тапу на дату); шторка «Добавить задачу» (по FAB); шторка «Импорт расписания» (по иконке импорта в AppBar).
- **Свайпы/жесты:** раскрывающийся недельный календарь (`ExpandableWeekCalendar`) — потянуть вниз, чтобы развернуть месяц; перелистывание недель/дней — внутри виджетов календаря/таймлайна.
- **Планшет (≥600px):** сверху во всю ширину строка переключателя вида + тумблер раскладки; ниже две колонки — слева управление (дата+Today, недельная полоса `WeekStrip`, поиск), справа содержимое вида.

### Экран Goals / «Долгосрочные цели» (файл: app/lib/features/plan/goals_screen.dart)

- **Назначение** — долгосрочные цели по горизонтам (Месяц → Год → 5 лет → 10 лет) с шагами-чеклистом. Открывается из AppBar таба Plan (иконка-флажок), это отдельный маршрут `/goals`, НЕ таб.
- **Шапка (AppBar):** СВОЙ AppBar (экран сам Scaffold). Слева — стандартная кнопка «назад» (push-маршрут). По центру/слева — заголовок «Цели» (`plan.goals_screen_title`). Справа действий нет.
- **Тело сверху вниз:**
  - Если целей нет — **пустое состояние** по центру: иконка-флажок (64dp) + текст-подсказка.
  - Если есть — скролл-список, **сгруппированный по горизонтам**: для каждого горизонта заголовок секции (Месяц / Год / 5 лет / 10 лет) и под ним карточки целей.
  - **Карточка цели** (`ExpansionTile`, разворачивается): заголовок цели, под ним прогресс-бар выполнения + текст «N из M шагов»; справа в свёрнутом виде — иконка **корзины** (удалить цель) и шеврон разворота `expand_more`. В развёрнутом виде — список **шагов-чекбоксов**, у каждого шага справа 2 иконки: **«запланировать сегодня»** (`today_outlined` — добавляет шаг задачей в Today) и **корзина** (удалить шаг); внизу — поле ввода нового шага + иконка `+` справа.
- **Кнопки и контролы:**
  - Удалить цель — иконка корзины в строке цели (с диалогом подтверждения).
  - Чекбокс шага — отметить выполненным.
  - «Запланировать сегодня» + удалить — иконки в строке каждого шага.
  - Поле + кнопка `+` — добавить шаг.
- **FAB (плавающая «+»):** ДА. Круглый FAB `+` снизу-справа → открывает **диалог создания новой цели** (`_NewGoalDialog`): поле названия + выбор горизонта чипами (Месяц/Год/5 лет/10 лет) + кнопки «Отмена» / «Создать». Позиция FAB настраивается (`fabPositionProvider`).
- **Нижние шторки / диалоги:** диалог создания цели (по FAB); диалог подтверждения удаления цели (по корзине, с danger-кнопкой ember-цвета); снэкбар «шаг удалён» с кнопкой «Отменить» (восстановление шага).
- **Свайпы/жесты:** **свайп влево по строке шага** удаляет шаг (`SwipeToDelete`) — дублирует кнопку-корзину, с тем же undo-снэкбаром.

### Экран Онбординг (файл: app/lib/features/onboarding/onboarding_screen.dart)

- **Назначение** — первый запуск: выбор языка + 3 вводных слайда о сути продукта, затем переход к входу (`/auth`). Показывается, пока не выставлен флаг `onboarding_done`.
- **Шапка (AppBar):** AppBar нет. Вместо него вверху-справа — кнопка **«Skip»** (текстовая, приглушённый цвет), видна на всех страницах, сразу завершает онбординг и ведёт на `/auth`.
- **Тело сверху вниз:** структура `Column` на весь экран:
  1. Сверху-справа — кнопка «Skip».
  2. **Область страниц** (`PageView`, листается горизонтально) — всего 4 страницы:
     - **Страница 0 — выбор языка:** по центру иконка `language_rounded` (64dp), заголовок, ниже скролл-**список из 12 языков**; каждый язык — кликабельная карточка (рамка, при выборе accent-обводка + галочка `check_circle`). Тап по языку сразу выставляет локаль и листает дальше.
     - **Страницы 1–3 — вводные слайды (value slides):** по центру маскот Kai (96dp, только на первом слайде) или иконка слайда (флажок / рассвет / книга, 64dp); ниже крупный заголовок (slug serif) и текст-подзаголовок. Слайды: «Главное не ускользнёт» / утренний-вечерний разбор / дневник (по смыслу иконок).
  3. **Dot-индикаторы** по центру — 4 точки, активная вытянута в «пилюлю» (24dp) accent-цветом.
  4. **Нижняя панель** (отступы 24dp, снизу 40dp):
     - На странице языка (0) — скрыта (переход идёт тапом по языку).
     - На слайдах 1–3 — слева квадратная кнопка **«назад»** (`arrow_back_rounded`, 52×52, только если есть предыдущая страница) + широкая **нижняя кнопка** (FilledButton, во всю оставшуюся ширину) с текстом **«Next»**, а на последнем слайде — **«Get started»** (завершает онбординг).
- **Кнопки и контролы:**
  - «Skip» — верх-право, пропустить весь онбординг.
  - Кнопки выбора языка — карточки-строки на странице 0.
  - «Назад» — квадратная кнопка снизу-слева (слайды 1–3).
  - «Next» / «Get started» — широкая нижняя кнопка (CTA), снизу.
- **FAB:** нет.
- **Нижние шторки / диалоги:** нет.
- **Свайпы/жесты:** горизонтальный свайп между страницами (`PageView`); навигация также кнопками Next/назад.

### Экран «Здоровье» (хаб) (`app/lib/features/health/health_screen.dart`)

**Назначение.** **Сводка здоровья** (не хаб-с-плитками): всегда видны трекер воды и трекер сна; ниже — компактные плитки-входы **только для включённых режимов-флагов** (Еда / Тренировки / Медитация / Дыхание). Флаги задаются в Профиле → «Поведение» (`feature_modes_provider.dart`), по умолчанию выключены; если все выключены — экран = Вода + Сон.

> ⚠️ **Список «10 плиток» ниже (п.4) УСТАРЕЛ** после эпика «план как позвоночник». В коде сейчас рендерятся только до 4 плиток за флагами (Food/Workouts/Meditation/Breathing). **Фокус, Зарядка, Экранное время, Осанка, Привычки, Совместная учёба — больше НЕ плитки на этом экране** (фокус/зарядка/экранное-время достигаются из плана; осанка — тумблер в Профиле; привычки и co-study выведены из навигации). Источник истины — SPEC §C5/§C8b.

**Шапка.** Своего AppBar НЕТ — это вкладка внутри общей нижней навигации. Вместо шапки первый элемент тела — крупный заголовок «Здоровье» (`health.title`, серифный display-шрифт, 32sp). Аватар/профиль приходит из общего каркаса вкладок, не из этого файла.

**Адаптив.** При ширине ≥600px (планшет) — двухколоночный режим: карточки Вода и Сон в один ряд по 50%, ниже плитки сеткой 2 в ряд. На телефоне (<600px) — одна колонка, всё списком (ListView, поля 24dp слева/справа, отступ снизу 96dp под нижнюю навигацию).

**Тело сверху вниз (телефон):**

### Сводка способов «как добавить еду» (для редизайна)

7. **ИИ-меню дня** (premium) — отдельная кнопка «Собрать ИИ» на экране.
8. **Из рецепта** — отдельный экран рецепта → «Записать рецепт».

## H. Планшет и веб — текущая адаптивность и пробелы для редизайна

> Источник: чтение кода `app/lib` (Grep + Read), ветка `night/2026-06-25-...`.
> Дата: 2026-06-26.

### H.1 Есть ли адаптив (краткий вердикт: ЧАСТИЧНО)

**ЧАСТИЧНО — да, но только в каркасе навигации и на 4 главных вкладках.**

Существует единый брейкпойнт-файл `app/lib/core/utils/breakpoints.dart`:
- `Breakpoints.tablet = 600` (строка 4)
- `Breakpoints.desktop = 900` (строка 5) — **объявлен, но НИГДЕ не используется** (метод `isDesktop`, стр. 10-11, не вызывается ни разу; Grep по `isDesktop`/`Breakpoints.desktop` дал 0 совпадений в коде, кроме самого определения). То есть «десктопного» (3-колоночного, очень широкого) режима НЕТ — есть только бинарное «телефон / ≥600».

Адаптив реализован ровно в 5 местах:
- каркас навигации `core/router/scaffold_with_nav_bar.dart` (LayoutBuilder, стр. 37-44);
- 4 главные вкладки: `today_screen.dart` (стр. 134), `plan_screen.dart` (стр. 74), `health_screen.dart` (стр. 118-125, LayoutBuilder), `diary_screen.dart` (стр. 157-164, LayoutBuilder).

Всё остальное (Профиль, Онбординг, Food/еда, Paywall, под-экраны Health — медитация, разминка, вода-фуллскрин, Цели, настройки, диалоги/боттом-шиты добавления задачи) адаптивных веток НЕ имеет — рендерится одноколоночный «телефонный» макет, растянутый на любую ширину.

### H.2 Навигация на планшете/широком экране (что происходит по факту)

Реализовано корректно в `scaffold_with_nav_bar.dart`:
- `LayoutBuilder` → при `constraints.maxWidth >= 600` вызывается `_buildWideLayout` (стр. 68-151): слева **NavigationRail** (`labelType: all`, аватар профиля в `leading`, стр. 108-110), вертикальный разделитель, справа `Expanded(child: navigationShell)`. Нижнего таб-бара нет.
- При ширине < 600 — `_buildMobileLayout` (стр. 155-228): обычный M3 `NavigationBar` снизу + аватар профиля в `leading` AppBar.
- Контекстные действия вкладки Plan (Цели, Импорт) есть в обоих режимах (`_planActions`, стр. 49-64).

Вывод: смена нижний таб-бар ↔ NavigationRail работает «из коробки» на ≥600px. Это единственная по-настоящему доведённая часть адаптива. Заметка в коде (стр. 106-107): NavigationRail требует `elevation > 0` иначе крэш — учтено.

### H.3 Ширина контента на больших экранах (растягивается или ограничена)

**В основном РАСТЯГИВАЕТСЯ на всю ширину — глобального ограничения maxWidth НЕТ.**

- В каркасе wide-layout контент идёт через `Expanded(child: navigationShell)` (стр. 147) — **без** `ConstrainedBox`/`Center`/`maxWidth`. Значит на широком мониторе (веб на десктопе) контент тянется от рейла до правого края экрана.
- Главные вкладки на планшете делают **2 колонки** (это спасает от сверхдлинных строк на ~600-900px, но не на FullHD):
  - Today `_buildTabletLayout` (стр. 247-345): `Row` из двух `Expanded` (левая — Kai/кольцо/серия/обзоры, правая — список задач), `padding: 24`. Каждая колонка — `Expanded`, без `maxWidth`.
  - Plan `_buildTabletLayout` (стр. 150+): полноширинная строка переключателя видов + 2 колонки (week strip flex:1 / таймлайн flex:2).
  - Diary `_buildTabletLayout` (стр. 186+): 2 колонки `Expanded` (форма / инсайты).
  - Health `_buildTabletLayout` (стр. 120-121): двухколоночная сетка карточек.
- Точечные `maxWidth` есть лишь локально и не системно: auth `auth_screen.dart:172` и `forgot_password_screen.dart:107` (420), food `food_screen.dart:1165/1171` (1024), paywall `paywall_screen.dart:536` (200, это про элемент), плюс речевые пузыри Kai. То есть карточки логина центрируются (хорошо), а большинство экранов — нет.

Итог: на планшете в портрете терпимо (2 колонки), но на вебе/широком десктопе строки текста и карточки растягиваются на всю ширину без читаемого предела (~1000-1200px) — типографически плохо.

### H.4 Веб-специфика (kIsWeb-ветки, что отличается)

`kIsWeb` используется ТОЛЬКО для отключения нативных возможностей, НЕ для лэйаута:
- Голосовой ввод (микрофон) скрыт на вебе: `core/widgets/voice_text_field.dart:48` (`_canShowMic = !kIsWeb`) и `today/widgets/add_task_sheet.dart:273`.
- Локальные уведомления полностью выключены на вебе: весь `services/notifications/notification_service.dart` (десятки `if (kIsWeb) return;` — стр. 37, 75, 82, 112, 152, 185, 213, 277, 299, 337, 365, 466).
- ОС-виджет домашнего экрана не поддерживается: `services/widget/widget_service.dart:82`.
- Видео-вложения на вебе не поддерживаются: `add_task_sheet.dart:896` (показывает снек «не поддерживается»), плюс восстановление потерянного вложения только Android `:874`.

Hover/курсоры есть, но НЕ через kIsWeb, а через `MouseRegion` (срабатывает только там, где есть мышь):
- `core/animations/pressable.dart:63-77` — приподнятие на hover (−2px);
- `plan/widgets/time_grid.dart:2012-2049` — курсор resize (resizeUpDown) на ручках блоков.

Чего на вебе НЕТ:
- Никакой веб-специфичной раскладки, отступов, ограничения ширины, обработки прокрутки/скроллбара, клавиатурной навигации, фокус-стилей. Веб = тот же мобильный/планшетный макет по ширине окна.
- `isDesktop` (≥900) не используется → на большом окне браузера не включается ничего сверх «планшетного» 2-колоночного режима.

### H.5 Ориентация (landscape)

**Обработки ориентации НЕТ вообще.**
- Grep по `SystemChrome`, `setPreferredOrientations`, `OrientationBuilder`, `Orientation.`, `landscape` → **0 совпадений** в `app/lib`.
- Ориентация не залочена (приложение свободно крутится) и нигде не учитывается явно.
- Следствие: всё завязано исключительно на **ширину**. Телефон в альбомной ориентации (ширина ~700-900px) автоматически попадёт в `isTablet`-ветку и покажет планшетный 2-колоночный макет/NavigationRail — при этом по высоте экран очень низкий. Это потенциально кривой кейс (две колонки на коротком альбомном телефоне), который дизайн нигде не учитывает.

### H.6 ВЫВОД ДЛЯ РЕДИЗАЙНА — где пробелы

Что СДЕЛАНО (можно опереться):
- Брейкпойнт-инфраструктура (`Breakpoints`, 600/900) и переключение нижний бар ↔ NavigationRail на ≥600 — работает.
- 4 главные вкладки имеют осмысленные планшетные 2-колоночные макеты.
- Нативные веб-ограничения (мик/уведомления/виджет/видео) аккуратно отключены.

Что НЕ адаптировано и что дизайнеру надо спроектировать заново:

1. **Десктоп/широкий веб (≥900-1200px) не спроектирован.** `Breakpoints.desktop` объявлен, но не используется. Нет третьего, «широкого» режима. На FullHD-мониторе всё либо тянется на всю ширину, либо остаётся 2 колонки с гигантскими полями. → Нужен дизайн desktop-режима: либо max-ширина контента (читаемый предел ~1100-1200px с центрированием), либо полноценная 3-зонная раскладка (рейл + контент + контекстная панель).

2. **Нет глобального ограничения ширины контента.** Каркас (`scaffold_with_nav_bar.dart:147`) и большинство экранов растягивают контент без `maxWidth`. → Дизайн должен задать единое правило «читаемой колонки» и где её центрировать.

3. **Не-табовые экраны вообще не адаптивны** (телефонный одноколоночник на любой ширине): Профиль, Онбординг (`setup_flow.dart`), Food/еда, Paywall, Цели, под-экраны Health (медитация, разминка `warmup_screen`, вода-фуллскрин), боттом-шиты добавления задачи. На планшете/вебе они выглядят как узкая телефонная лента по центру растянутого экрана либо растянуты во всю ширину. → Нужны планшет/веб-макеты для каждого из них (особенно Онбординг и Paywall — это «лицо» на вебе).

4. **Ориентация landscape не учтена.** Width-only брейкпойнты → телефон в альбоме ошибочно получает «планшетный» режим на низком экране. → Дизайнеру задать поведение для коротких/альбомных экранов (учитывать высоту, а не только ширину; например требовать минимум высоты для 2 колонок).

5. **Веб-эргономика отсутствует:** нет проектирования под мышь/клавиатуру (hover-стили есть лишь точечно через `MouseRegion`), нет фокус-колец, скроллбаров, плотности под десктоп, поведения при ресайзе окна. Голосовой ввод на вебе скрыт — на этих экранах нужно компенсировать UX (текстовый ввод как основной).

6. **Брейкпойнт 600 один на всё.** Граница «телефон/планшет» = 600, без промежуточных состояний. → Возможно нужна шкала (например compact <600, medium 600-900, expanded 900-1200, large >1200) с разной плотностью.

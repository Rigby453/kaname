# Видение: «Говоришь планеру, а не настраиваешь руками» (AI voice-first planning)

> Большая концептуальная функция (запрос пользователя 2026-07-01). НЕ кодить сразу — продумать.
> Это главный платный hook (усиливает ценность Premium 399₽/$5).

## Суть
Пользователь не настраивает задачи кнопками — он **говорит**, а ИИ понимает, структурирует и
расставляет. Меньше кнопок, больше контекста у ИИ, план «сам собирается».

## Три столпа

### 1. Онбординг — голосовой «брейн-дамп» контекста
При первом входе — большое поле с **голосовым вводом**, где человек МАКСИМАЛЬНО подробно описывает:
- задачи на сегодня-завтра;
- свои **сферы и структуру жизни** (учёба, работа, здоровье, отношения, проекты…);
- что сейчас мешает / что важно.

ИИ из этого строит **профиль контекста** (сферы + постоянные обязательства + ограничения) и
**грамотно расставляет первые задачи**. Профиль СОХРАНЯЕТСЯ и подаётся во все будущие ИИ-вызовы
(богаче контекст → умнее перенос/советы). Требует сильной модели (см. ниже).

### 2. Голосовой быстрый ввод задачи
Появилась задача → одна кнопка → говоришь «в 7 завтра встреча с Гайдляновым» → ИИ сам определяет
время, название, тип (событие/встреча), приоритет, длительность, повтор → создаёт и ставит в план.
Минимум кнопок, пользователь НЕ настраивает задачу руками. Надстройка над голосом + текущим
парсером, но шаг понимания делает ИИ (не только regex).

### 3. Ритуал дня: утро отдаёт / вечер собирает
- **Утро:** ИИ отдаёт план на сегодня и ближайшие дни (расставленный).
- **Вечер:** пользователь голосом описывает, что сделал и как потратил время → ИИ **сам определяет,
  какие задачи выполнены, и закрывает их** (меньше ручного закрытия), плюс выявляет паттерны/ошибки
  («почему план не сработал» — ядро продукта «понять, почему планы рушатся»).
Это объяснить в онбординге (чего ждать каждый день).

- Провайдер ИИ умеет Gemini И Claude Sonnet (`backend/src/ai/provider.ts:170 claude-sonnet-4-6`) — смена модели = конфиг.
- Утренний/вечерний разбор (rule-based) + дневник (вечерняя рефлексия) уже есть.
ЧЕГО НЕТ: профиль «сфер»/контекста; ИИ-структурирование голоса (сейчас regex); вечернее авто-закрытие из речи.

## Ключевые решения (обсудить)
1. **Модель.** Для понимания свободной речи flash-lite слаб (и сейчас бьётся в квоту). Нужен сильный:
   Gemini 2.5 Pro ИЛИ Claude Sonnet 4.6. Дороже → строго Premium. Какой и подтвердить «только платно».
2. **Авто vs подтверждение.** Полностью авто-расстановка ИЛИ всегда показывать превью с подтверждением/
   drag (= уже одобренный «утренний разбор-превью»). Рекоменд.: на старте — подтверждение, заработав
   доверие → больше авто.
3. **Вечернее авто-закрытие из речи.** Насколько доверять ИИ закрывать задачи по устному описанию.
   Риск ложного закрытия. Рекоменд.: ИИ ПРЕДЛАГАЕТ, пользователь подтверждает батч одним тапом.
4. **Модель «сфер».** Как хранить сферы как редактируемый контекст (профиль? теги? связь с Целями?).
   Стыкуется с «формами задач» (fuzzy-tasks) и Goals.
5. **Приватность.** Подробные личные брейн-дампы уходят провайдеру ИИ → нужно согласие + честное
   описание в Terms/Privacy (#13). Возможно — пометка «не отправлять чувствительное».
6. **Латентность/стоимость.** Большие голос→ИИ вызовы медленные и дорогие → прогресс-UX, кэш, батч.

## Предлагаемое поэтапно
- **Фаза A (близко, на текущих кирпичах):** голосовой быстрый ввод задачи → ИИ-структурирование
  (одна кнопка → говоришь → ИИ-задача с подтверждением) + утренний разбор-превью (уже greenlit) +
  вечерняя голосовая рефлексия в дневник.
- **Фаза B:** онбординг брейн-дамп → профиль контекста, подаётся во все ИИ-вызовы.
- **Фаза C:** вечернее авто-закрытие задач из речи + выявление паттернов («почему план не сработал»).
Каждая фаза — отдельный платный hook.
